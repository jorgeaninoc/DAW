¿En qué consiste el acceso basado en roles?

El acceso basado en roles permite el acceso a distintos usuarios a un sistema. A cada usuario se le asigna un rol que contiene un conjunto de permisos. Con base en esos permisos se define la interfaz que tendrá el usuario, es decir los permisos que se desplegaran en la pantalla. Segun la documentación de ORACLE es una función de seguridad para controlar el acceso de usuarios a tareas que normalmente están restringidas al superusuario.

Investiga y describe 6 sistemas, 3 conocidos que empleen RBAC y 3 desconocidos que no, junto con su funcionamiento general.

Trello
Un gestor de proyectos en el que puedes incluir la lista de actividades que se deben realizar y editarlas para poner checklist, fecha de entrega, responsable, etcétera.
Todos tienen el mismo privilegio.
Facebook
Red social en la que el usuario tiene el mismo privilegio que cualquier otro usuario que acaba de crear su cuenta. Se tiene la posibilidad de quitar o limitar a un usuario el permiso de ver el contenido que publicas.
Todos tienen el mismo privlegio.
Netflix
Aplicación Móvil, y Web que permite ver películas y series. En el menú de inicio puedes editar la cuenta de cada usuario y modificar lo que pueden ver (categorías). Aunque todas las cuentas pueden ser accesadas con la misma contraseña.
Freedcamp
Sitio web para gestionar la administración de proyectos. El creador de un proyecto puede agregar usuarios y modificar su rol (el cual cuenta con x permisos). En el sitio pueden compartir documentos agregar eventos al calendario chatear y más.
BlackBoard
Plataforma para alumnos y profesores donde se publica contenido de los cursos.
CECEQ (próximamente)
Un sistema que emplea el modelo RBAC para controlar el acceso a los usuarios de la biblioteca delegando los permisos que les son otorgados. De esa forma mejorar la seguridad.

Beneficios y desventajas de este modelo.

VENTAJAS
-Es fácil ver los permisos del mismo para todos los sujetos.
– Es fácil revocar todos sus accesos, reemplazando su lista de acceso por una vacía.
– Es fácil darlo de baja, borrando su lista de acceso.
-Flexible y adaptable a muchos sistemas y aplicaciones.
-Ampliamente usado
DESVENTAJAS
– Dificultad de establecer los roles y definir sus alcances.
-Más complejo de programar
- Usuarios con menor privilegios se sienten con menos poder.

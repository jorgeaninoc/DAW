¿Por qué es una buena práctica separar el controlador de la vista?

Separar la lógica de la interfaz de usuario facilita la funcionalidad, mantenibilidad y escalabilidad del sistema, de forma comoda y sencilla, a la vez que podemos mezclar lenguajes de programación en el mismo código.

Aparte de los arreglos $_POST y $_GET, ¿qué otros arreglos están predefinidos en php y cuál es su función?

  1. $GLOBALS: Contiene todas las variables globales
  2. $_SERVER: Contiene información como cabeceras, rutas y ubicaciones de script.
  3. $_FILES: Elementos subidos al script a través del metodo POST.
  4. $_COOKIE: Contiene variables pasadas al script actual a través de cookies.
  5. $_SESSION: Contiene variables de sesión disponibles.
  6. $_REQUEST: Contiene el contenido de $_GET, $_POST y $_COOKIE.
  7. $_ENV: Variables de entorno pasadas al script actual.

¿Qué es XSS y cómo se puede prevenir?

  • Es un tipo de ataque en donde el hacker inyecta codigo malicioso en el cliente para que afecte el output de tu página web. Las aplicaciones que no tienen la manera de escapar del output al ataque son vulnerables.
  • Se previene utilizando funciones como htmlspecialchars() o htmlentities() que previenen el output de la pagina. Ambas funciones de php convierten chars problematicos en entidades de HTML haciendo que el codigo inyectado sea inofensivo.
